# 🔗 URL Shortener

![Tests](https://github.com/AVALON4231/url-shortener/actions/workflows/tests.yml/badge.svg)
![Python](https://img.shields.io/badge/Python-3.11+-3776AB?logo=python&logoColor=white)
![FastAPI](https://img.shields.io/badge/FastAPI-0.100+-009688?logo=fastapi&logoColor=white)
![License](https://img.shields.io/badge/License-MIT-green)

Персональный сервис сокращения ссылок с JWT-аутентификацией, защитой от злоупотреблений, автоматическим извлечением заголовков страниц и историей переходов. Доступен через веб-интерфейс и API (Swagger).

Проект создан в учебных целях с использованием AI-ассистента, но решает реальные задачи: помогает организовать закладки, отслеживать переходы и делиться ссылками без лишнего шума.

---

## ✨ Ключевые особенности

| Особенность | Описание |
|---|---|
| 🔒 Приватность | Каждый пользователь видит только свои ссылки и статистику |
| 🧠 Умный контекст | Сервис автоматически извлекает `<title>` страницы и показывает его в истории |
| 📊 Статистика | Счётчик переходов и дата создания для каждой ссылки |
| 🛡️ Rate Limiting | Защита от спама: ограничение числа запросов с одного IP |
| ✅ Валидация URL | Принимаются только `http://` и `https://` ссылки |
| 🚫 Защита от SSRF | Блокировка редиректов на внутренние адреса сети |
| 🗑️ Управление | Удаление ненужных ссылок прямо из истории |
| 🔄 Адаптивный ответ | Редирект для браузеров и JSON для API-клиентов |
| 🗄️ Гибкое хранилище | SQLite локально, PostgreSQL на продакшене — без изменений в коде |
| ⚙️ CI/CD | Тесты через GitHub Actions, деплой на Render, документация Swagger |

---

## 🚀 Функционал

- Регистрация и вход по любому логину (JWT-токены)
- Создание коротких ссылок с привязкой к аккаунту
- Автоматическое копирование в буфер обмена после сокращения
- Редирект на оригинальный URL с подсчётом кликов
- История ссылок с заголовком страницы, счётчиком переходов и датой
- Удаление своих ссылок с красивым подтверждением
- Статистика по конкретной ссылке через API
- Автоматическая документация API (Swagger UI)
- Современный веб-интерфейс (модульный JS, Bootstrap 5)

## ⚡ Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/AVALON4231/url-shortener.git
cd url-shortener
```

### 2. Создайте виртуальное окружение

```bash
python -m venv venv
source venv/bin/activate      # Linux / Mac
venv\Scripts\activate         # Windows
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Задайте переменные окружения

| Переменная | Обязательно | Описание |
|---|---|---|
| `SECRET_KEY` | Для продакшена | Случайная строка для подписи JWT. Без неё — используется небезопасный ключ по умолчанию, в консоль выводится предупреждение |
| `DATABASE_URL` | Нет | Строка подключения к PostgreSQL. Без неё — используется SQLite |

Сгенерировать безопасный `SECRET_KEY`:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

Пример `.env` для локальной разработки:
```env
SECRET_KEY=ваш-секретный-ключ
DATABASE_URL=postgresql://user:password@localhost:5432/shortener
```

### 5. Запустите сервер

```bash
uvicorn main:app --reload
```

После запуска:
- Веб-интерфейс: [http://127.0.0.1:8000](http://127.0.0.1:8000)
- Документация API: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

## 📡 API

| Метод | Путь | Доступ | Описание |
|---|---|---|---|
| `POST` | `/register` | Публичный | Регистрация нового пользователя |
| `POST` | `/login` | Публичный | Вход, возвращает JWT-токен |
| `POST` | `/shorten` | Авторизация | Создать короткую ссылку (body: `{"url": "..."}`) |
| `GET` | `/my-links` | Авторизация | История своих ссылок |
| `DELETE` | `/my-links/{code}` | Авторизация | Удалить свою ссылку |
| `GET` | `/stats/{code}` | Авторизация | Статистика по ссылке |
| `GET` | `/{code}` | Публичный | Редирект на оригинальный URL |

> Для авторизованных эндпоинтов передавайте заголовок:
> `Authorization: Bearer <токен>`

---

## 🛡️ Безопасность

- Пароли передаются в теле запроса (JSON) — не попадают в URL и логи сервера
- Пароли хранятся только в хешированном виде (sha256_crypt)
- Короткие коды генерируются через `secrets` — криптографически стойкий генератор
- Rate limiting: макс. 5 регистраций / 10 попыток входа / 10 сокращений в минуту с одного IP
- Принимаются только `http://` и `https://` URL — блокируются `javascript:`, `file://` и прочее
- Защита от SSRF: URL с внутренними IP-адресами (`127.x`, `10.x`, `192.168.x`) отклоняются
- Каждый пользователь может видеть и удалять только свои ссылки
